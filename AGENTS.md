# AGENTS.md

This repository is a **Flatpak packaging manifest** for WeylusCommunityEdition, not the app source.

## What this repo contains

- `io.github.electronstudio.WeylusCommunityEdition.json` — Flatpak build manifest.
- `build.rs.patch` — downstream patch to force `--offline` for `pnpm install` / `npm install` inside the Flatpak build sandbox.
- `generated-sources-cargo.json` — vendored Rust crates for `flatpak-builder` offline builds.
- `generated-sources-node.json` — vendored npm/pnpm packages for `flatpak-builder` offline builds.

## Upstream source

Source is fetched from upstream during build:
```
https://github.com/electronstudio/WeylusCommunityEdition/archive/refs/tags/2024.8.1.tar.gz
```

## Build environment

- **Runtime:** `org.freedesktop.Platform` 23.08
- **SDK extensions:** `org.freedesktop.Sdk.Extension.rust-stable`, `org.freedesktop.Sdk.Extension.node18`
- **Pre-built deps:** x264, ffmpeg (system shared libs)
- **Build command:** `cargo b --features ffmpeg-system --release`

## Flatpak-specific constraints

- `CARGO_NET_OFFLINE=true` and `--offline` npm/pnpm flags are required; network access is disabled inside the build.
- `RUSTFLAGS="-A dead_code -A unused_imports -L /app/lib"` suppresses warnings and links system libs.
- The patch (`build.rs.patch`) is applied to upstream `build.rs` before building to inject `--offline` flags.
- `generated-sources-*.json` files must be kept in sync with upstream dependency versions; they are auto-generated by flatpak-builder or `flatpak-builder-tools` scripts.

## Typical edit workflow

1. Update the upstream archive URL + checksum in the manifest when a new release drops.
2. Regenerate `generated-sources-cargo.json` and `generated-sources-node.json` with flatpak-builder tools (e.g., `flatpak-cargo-generator.py`, `flatpak-node-generator`) based on the new upstream Cargo.lock / package-lock.json.
3. Verify the patch still applies cleanly to the new upstream `build.rs`.
